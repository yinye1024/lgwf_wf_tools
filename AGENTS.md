# 仓库协作指引

## 语言要求

- 默认使用中文进行说明、注释和面向人的文档编写。
- Markdown 文件必须优先使用中文，包括标题、段落、列表、说明文字和验收步骤。
- 代码标识符、命令、路径、配置键、API 名称和引用的外部专有名词可以保留英文原文。
- 如果需要引用英文资料，应先用中文概括结论，再按需保留必要的英文原句或术语。

## 文档风格

- 文档内容应直接、清晰、可执行，避免空泛描述。
- 新增或修改 `README.md`、`SKILL.md`、`AGENTS.md` 以及其他 `.md` 文件时，优先检查是否符合中文表达要求。
- 若现有文件是英文，可以在修改相关段落时逐步改为中文；不要为了无关任务大规模重写整份文档。

## 变更约束

- 遵循现有目录结构和命名方式。
- 不要改动与当前任务无关的文件。
- 对插件、skill 或 workflow 的行为变更，应同步更新相应中文文档和最小验证方式。
